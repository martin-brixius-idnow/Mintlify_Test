---
title: Biometric Enrollment
description: Manages the facial biometric enrollment of a user.
---

## Overview

This Step node manages facial biometric enrollment and supports silent enrollment using an external image source via an IDV bridge.

---

## Configuration Options

| Parameter   | Type    | Required | Description |
|------------|---------|----------|-------------|
| `maxRetries` | Integer | No       | Defines the maximum number of retries the underlying service can attempt in case of a capture failure or image quality validation failure. If this option is not provided, a default value of 3 is applied. |

---

## Input Data

Input requirements for this node.

| DataBlock | Mandatory | Description |
|----------:|:---------:|-------------|
| `UserReference` | Optional | Contains the unique subject identifier or the user alias (mail/tel etc). |
| `BiometricSamples` or `DocumentImages` | Conditional | An image source is required: `BiometricSamples` (priority) or `DocumentImages`. |

---

## Output Data

DataBlocks produced per verdict.

| Scenario         | DataBlocks Produced      | Description |
|------------------|-------------------------|-------------|
| accepted        | `AuthenticatorCredential` | The enrollment was successful. The authentication means has been created and is associated with the subject.

A user identifier is generated by the node if the client does not provide a value (i.e., no DataBlockType.UserReference provided as input). |
| rejected         | ∅                    | The enrollment has failed after all retry attempts. |
| already_enrolled | ∅                    | The user is already enrolled. |
