Learn how to set up authentication and obtain an API token to connect to Trust Platform APIs.

IDnow uses OAuth 2.0 with the client credentials grant type for API authentication.

This guide shows you how to:

1. Get your API client credentials from the IDnow Dashboard
2. Obtain an access token using your credentials
3. Make an authenticated API request

---

## Get credentials

To authenticate against Trust Platform APIs, you need to obtain your API client credentials (`client_id` and `client_secret`).

<Info>
  For detailed instructions, refer to <a href="https://idnow.mintlify.app/Development_Resources/Manage" target="_blank" rel="noopener noreferrer">Manage API Credentials</a>.
</Info>

---

## Obtain access token

Use your client credentials to request an access token. Trust Platform supports two client authentication methods:

***Request parameters***

| Parameter       | Description                                                                 | Required |
|-----------------|-----------------------------------------------------------------------------|----------|
| `grant_type`    | OAuth 2.0 grant type. Use `client_credentials` for server-to-server auth.  | Yes      |
| `client_id`     | Your API client ID from the IDnow Dashboard.                                | Yes      |
| `client_secret` | Your API client secret from the IDnow Dashboard.                            | Yes      |
| `scope`         | The scopes required for your API requests (if applicable).                  | Optional |

***Option A***: Explicit credentials (`client_secret_post`)

Include your credentials directly in the request body:

```bash
curl --request POST https://<your-idnow-auth-server>/token \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data 'grant_type=client_credentials' \
  --data 'client_id=<your-client-id>' \
  --data 'client_secret=<your-client-secret>' \
  --data 'scope=<required-scopes>'
```

***Option B***: HTTP Basic Authentication (`client_secret_basic`)

Base64 encode your credentials in the format `client_id:client_secret` and include in the Authorization header:

```bash
curl --request POST https://<your-idnow-auth-server>/token \
  --header 'Authorization: Basic <BASE64_ENCODED_CREDENTIALS>' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data 'grant_type=client_credentials' \
  --data 'scope=<required-scopes>'
```

***Response***

If successful, you will receive an access token:

```json
{
  "access_token": "eyJh...QifQ.eyJ...hIOw",
  "expires_in": 3600,
  "token_type": "Bearer",
  "scope": "<granted-scopes>"
}
```

> **Note:** The default access token lifetime is typically 3600 seconds (1 hour). Cache tokens appropriately to avoid unnecessary requests.

## Make an API request

Now that you have a valid access token, you can make requests to IDnow APIs. Include the access token in the `Authorization` header using the `Bearer` scheme:

```bash
GET /api/v1/<endpoint> HTTP/1.1
Host: <your-idnow-api-host>
Accept: application/json
Authorization: Bearer eyJh...QifQ.eyJ...hIOw
```

***Example request***

```bash
curl --request GET https://<your-idnow-api-host>/api/v1/identifications/<id> \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer eyJh...QifQ.eyJ...hIOw'
```
