## Overview

API keys securely authenticate your client applications with Trustplatform. Each key has a **Public key ID** and a **Client secret**. You can create multiple keys, assign permissions, and manage them in IDnow Studio.

> **Never share client secrets. Do not store API keys in version control.**

---

## Manage keys

1. Log in to [Studio](https://hub.hub.dev.eu.idnow.io/auth/login).
2. Go to **Settings** > **DEVELOPER** > **API keys**.

![API keys management interface](https://raw.githubusercontent.com/martin-brixius-idnow/Mintlify_Test/main/images/Resources/API_Keys-Overview.png)

---

## Create key

1. Select **New API key**.
2. Enter a descriptive name.
3. Select permissions:

| Permission | Description |
|-----------|-------------|
| `Execute flow` | Initiate and execute identity verification flows |
| `Read API resources` | Retrieve data from the API |
| `Edit and create API resources` | Modify and create resources via the API |

> Select only needed permissions.

4. Select **Create API key**.

![Create API key dialog](https://raw.githubusercontent.com/martin-brixius-idnow/Mintlify_Test/main/images/Resources/API_Keys-Create.png)
*Create API key dialog*

---

## Save credentials

- Copy **Public key ID** and **Client secret** securely.

> **Copy the client secret now. You won't see it again.**  
> Lost secrets must be regenerated and updated in all apps.

- Confirm you have saved credentials.

---

## View key details

![Save API key](https://raw.githubusercontent.com/martin-brixius-idnow/Mintlify_Test/main/images/Resources/API_Keys-Save-Key.png)
*Save API key*

- See name, creator, creation date, last used, actions (view, delete).

---

## Edit key

1. Find your API key, select **View details**.
2. Edit name, permissions, or regenerate client secret.

> 'Public key ID' cannot be changed.

![Edit API key dialog](https://raw.githubusercontent.com/martin-brixius-idnow/Mintlify_Test/main/images/Resources/API_Keys-Edit.png)
*Edit API key dialog*

---

## Regenerate secret

1. Select **Regenerate**.
2. Confirm and copy new secret.

![Regenerate client secret dialog](https://raw.githubusercontent.com/martin-brixius-idnow/Mintlify_Test/main/images/Resources/API_Keys-Regenerate.png)
*Regenerate client secret dialog*

> **Important:** Old secret is revoked. Update all apps.

---

## Delete key

1. Find and select **Delete icon**.
2. Confirm deletion.

![Delete API key confirmation](https://raw.githubusercontent.com/martin-brixius-idnow/Mintlify_Test/main/images/Resources/API_Keys-Delete.png)
*Delete API key confirmation*

> **This action cannot be undone.** Update or disable all apps using this key before deletion.

---

## Security best practices

- Never share secrets.
- Don't store keys in version control.
- Use environment variables or secure storage.
- Grant minimum permissions.
- Remove unused keys.
- Regenerate secrets periodically.
- Use separate keys for each environment.
- Test rotation before applying to Live.
