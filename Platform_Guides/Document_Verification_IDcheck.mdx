## Overview

Automated instant identity verification using AI and Machine Learning. Guides users through interactive document capture and biometric verification with real-time processing via IDCheck.io.

---

## Key Features

- **Interactive Capture** - SDK-guided document and biometric collection
- **AI-Powered Analysis** - Automated fraud detection and data extraction
- **Manual Review Option** - Human operator verification when needed
- **Multi-Document Support** - ID, passport, permits, visas, licenses, health cards
- **Biometric Verification** - Selfie and video liveness detection
- **Flexible Consent** - SDK or customer-managed consent handling

---

## Configuration

### Capture Settings

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `documentTypes` | Array | **Yes** | - | Document types to accept: `ID`, `PASSPORT`, `RESIDENT_PERMIT`, `VISA`, `DRIVING_LICENSE`, `HEALTH_CARD` |
| `biometricSampleTypes` | Array | No | `[]` | Biometric capture: `SELFIE`, `VIDEO`. Empty array = disabled |
| `biometricConsentHandling` | String | No | `SDK` | Consent collection: `SDK` (handled during capture) or `CUSTOMER` (pre-collected) |

---

## Input Requirements

**No input DataBlocks required**

This node initiates the verification flow and collects data directly from the user.

---

## Verdicts & Outputs

Below is a full overview of all possible outcomes of the Document Verification process.

| **Scenario** | **DataBlocks Produced** | **Description** |
|--------------|--------------------------|------------------|
| **verified** | `DocumentVerification`, `BasicIdentity`, `ExtendedIdentity`, `DocumentData`, `DocumentImages`, `BiometricSamples`* | The document and biometric samples (if configured) were successfully processed and data was extracted. <br/><br/>*`BiometricSamples` is only produced if `biometricSampleTypes` was configured with one or more values and capture was successful.* |
| **fraud_detected** | `DocumentVerification` | Fraud was detected on the document or biometric samples (if configured), or they were no longer valid. No identity data is extracted. |
| **capture_failed** | `DocumentVerification` | The quality of the provided images did not allow for processing. |
| **default** | `DocumentVerification` | A technical or unexpected error occurred during processing. |
