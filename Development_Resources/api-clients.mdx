---

An API client securely authenticates your application with IDnow. You can manage multiple keys in the IDnow dashboard.

Each API client has a **Client ID** and a **Client secret**. Never share client secrets or store them in version control.**

---

## Manage keys

1. Log in to the IDnow dashboard at [hub.hub.dev.eu.idnow.io](https://hub.hub.dev.eu.idnow.io/auth/login).
2. Go to **Settings** > **Developer** > **API clients**.

![API keys management interface](/img/resources/api-keys-overview.png)
_API key details page_

---

## Create key

1. Select **New API key**.
2. Enter a descriptive name.
3. Select **Create API key**.


---

## Save API client details

1. Copy **Client ID** and **Client secret** securely.
2. Confirm you have saved them.


> **Copy the client secret now. You won't see it again.**  
> Lost secrets must be regenerated and updated in all apps.

---

## Edit key

1. Find your API key, select **View details**.
2. Edit name or regenerate client secret.

> 'Client ID' cannot be changed.

---

## Regenerate secret

1. Select **Regenerate**.
2. Confirm and copy new secret.

> **Important:** Old secret is revoked. Update all apps.

---

## Delete key

1. Find and select **Delete icon**.
2. Confirm deletion.

> **This action cannot be undone.** Update or disable all apps using this key before deletion.

---

## Security best practices

- Never share secrets.
- Don't store keys in version control.
- Use environment variables or secure storage.
- Remove unused keys.
- Regenerate secrets periodically.
- Use separate keys for each environment.
- Test rotation before applying to Live.
