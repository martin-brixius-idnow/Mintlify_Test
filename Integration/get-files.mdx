## Overview

Retrieve specific files (such as document images, PDFs, or other attachments) generated or uploaded during a session. Files are referenced by their unique file identifier.

## Endpoint

```http
GET /files/{fileId}
```

### Path parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `fileId` | string | Yes | The unique identifier of the file to retrieve |

## Request

No request body is required for this endpoint.

### Headers

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `Authorization` | string | Yes | Bearer token for authentication |

## Response

The response returns the requested file as a binary stream with appropriate content type headers.

### Response headers

| Header | Description |
|--------|-------------|
| `Content-Type` | MIME type of the file (e.g., `image/jpeg`, `application/pdf`) |
| `Content-Length` | Size of the file in bytes |
| `Content-Disposition` | Suggested filename for download |

## Example

```bash
curl https://api.idnow.io/v1/files/file_abc123xyz \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -o downloaded_file.jpg
```

## Use cases

- Retrieve document images (passport, ID card photos)
- Download verification reports
- Access uploaded supporting documents
- Retrieve facial biometric images

## Notes

- No schema changes are planned for this endpoint in V1
- All file access attempts are logged for compliance and auditability
- Files are stored securely and access requires proper authentication
- Large files are streamed to optimize performance
