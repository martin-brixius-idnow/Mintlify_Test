---
title: "Get flows"
description: "Retrieve a list of available flows configured for your environment"
---

Use this endpoint to retrieve all flows available in your environment. The response provides essential metadata for each flow.

## Endpoint

```http
GET /flows
```

## Request

No request body is required for this endpoint.

***Headers***

| Header          | Type   | Required | Description |
|-----------------|--------|----------|-------------|
| `Authorization` | String | Yes | Bearer token for authentication |
| `Content-Type`  | String | Yes | Must be `application/json` |

## Response

Returns an array of flow objects with minimal metadata.

```json
{
  "content": [
    {
      "flowID": "string",
      "flowName": "string",
      "flowDescription": "string"
    }
  ]
}
```

***Response parameters***

| Parameter         | Type   | Description |
|-------------------|--------|-------------|
| `content`         | Array  | Array of flow objects |
| `flowID`          | String | Unique identifier for the flow |
| `flowName`        | String | Display name of the flow |
| `flowDescription` | String | Description of the flow's purpose |

## Example

```bash
curl https://api.idnow.io/v1/flows \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

## Notes

- All flows configured for your environment are returned in a single response
