#### Initiate a new verification session for a specific flow

Create a new session to start a verification flow. The session begins in a `in progress` **sessionStatus**.

## Endpoint

```http
POST v1/flows/{flowId}/{environment}/sessions
```

**_Path parameters_**

| Parameter | Type   | Required | Description                                  |
| --------- | ------ | -------- | -------------------------------------------- |
| `flowId`  | String | Yes      | The unique identifier of the flow to execute |

## Request

```json
{
  "input": [
    {
      "type": "BasicIdentity",
      "name": "Max Schmidt",
      "familyName": "Schmidt",
      "givenName": "Max",
      "birthDate": "1985-08-22"
    },
    {
      "type": "DocumentImages",
      "documentType": "ID",
      "frontSide": "byte array",
      "backSide": "byte array"
    }
  ]
}
```

**_Request parameters_**

| Parameter | Type  | Required    | Description                |
| --------- | ----- | ----------- | -------------------------- |
| `input`   | Array | Conditional | Array of input data blocks |

## Response

```json
{
  "flowId": "string",
  "flowVersion": "string",
  "environment": "string",
  "sessionContext": {
    "sessionId": "123",
    "sessionStatus": "created",
    "createdAt": "2024-11-04T10:30:00Z"
  },
  "sessionToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**_Response parameters_**

| Parameter                      | Type   | Description                               |
| ------------------------------ | ------ | ----------------------------------------- |
| `flowId`                       | String | Flow identifier                           |
| `flowVersion`                  | String | Version of the flow being executed        |
| `environment`                  | String | Environment where the session was created |
| `sessionContext`               | Object | Session metadata                          |
| `sessionContext.sessionId`     | String | Unique session identifier                 |
| `sessionContext.sessionStatus` | String | Initial status: `created`                 |
| `sessionContext.createdAt`     | String | ISO 8601 timestamp of session creation    |
| `sessionToken`                 | String | JWT token used to initialize the Player   |

## Example

```bash
curl https://api.eu.platform.idnow.io/api/v1/flows/12345/live/sessions
  -X POST \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "input": [
      {
        "type": "BasicIdentity",
        "familyName": "Schmidt",
        "givenName": "Max",
        "birthDate": "1985-08-22"
      }
    ]
  }'
```

## Notes

- Session status is set to `in progress` immediately upon successful creation
- Input object format is analogous to the data block endpoint structure
- Supports both interactive (customer-involved) and non-interactive (headless) flows
- The `sessionToken` is a JWT that contains the session context and is used to initialize the Player
