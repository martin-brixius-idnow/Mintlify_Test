## Overview

The purpose of this step is to enable the display of a page within a workflow that presents users with a list of client-specified identity verification methods. Based on the choice made by the user, they are then directed to the step associated with the chosen method.

## Key Features

- **User-Driven Selection**: Displays a selection page in Player with available verification methods
- **Dynamic Ports**: Creates output ports dynamically based on configured methods
- **Method Assets**: Each method includes display assets (label, image, country)
- **Flexible Routing**: No constraints on the type of step that can be associated with transitions


## Configuration

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| methods | array | Yes | A list of unique string identifiers for the verification methods to be offered (e.g., doc_id, france_identite, eudi_wallet). Each string is used to name a corresponding dynamic output port. Minimum 1 method required. |

**Frontend Implications:**

Each method that can be defined as an option is associated with a set of assets intended for displaying a selection block, which contains the radio button for the selection:

- A label associated with the method (its title)
- An image displayed within the block
- The country to which this method is associated (e.g., "France" for France Identit√© or "Germany" for AusweisID)

## Example

```json
{
  "methods": [
    "doc_id",
    "france_identite",
    "eudi_wallet_verifier"
  ]
}
```

## Input data block

This step has no static input data block. It does not consume any data blocks.

## Output data block

This step does not produce any output data block.
